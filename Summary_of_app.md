# FillIt 앱: 보고서 및 발표 요약 (확장 버전)

## 앱 개요
**FillIt**은 사용자의 빈 시간 슬롯 동안 장소를 추천하는 Android 애플리케이션입니다. 카테고리, 이동 수단, 가격 범위에 대한 사용자 선호도를 활용하여 개인 맞춤형 추천을 제공합니다. 앱은 인증, 장소 데이터, AI 기반 기능에 Google 서비스를 통합합니다.

## 주요 기능
- **개인 맞춤형 추천**: 빈 시간 기간, 위치, 사용자 선택 선호도를 기반으로 장소를 제안합니다.
- **설정 관리**: 관심 카테고리(예: 카페, 전시, 체험, 관광지, 맛집, 쇼핑), 이동 수단 선호도(도보/자차), 가격 범위(무료 ~ 고가), 알림 설정을 구성할 수 있습니다.
- **검색 기능**: 키워드 기반 검색과 카테고리 기반 필터링을 지원합니다.
- **즐겨찾기 시스템**: 즐겨찾기 장소를 저장하고, AI 생성 키워드로 검색성을 향상시킵니다.
- **위치 통합**: 근처 추천을 위해 위치 권한을 요청하며, 비활성화 시 서울, 대한민국으로 기본 설정됩니다. GPS를 통해 사용자의 실제 위치를 동적으로 가져옵니다.
- **사용자 인증**: Google 로그인으로 사용자 프로필과 데이터 지속성을 관리합니다.

## 아키텍처
### 클라이언트 측 (Android)
- **프레임워크**: UI를 위한 Jetpack Compose, Kotlin 프로그래밍 언어.
- **아키텍처 패턴**: Repository 패턴과 함께 MVVM (Model-View-ViewModel).
- **데이터 저장소**:
  - 사용자 선호도(카테고리, 이동 수단, 가격, 알림)를 위한 Android DataStore.
  - 즐겨찾기와 생성된 키워드를 위한 Firebase Firestore.
- **의존성 주입**: API 클라이언트와 리포지토리 관리를 위한 Service Locator 패턴.
- **네트워킹**: Node.js 백엔드로의 API 호출을 위한 Retrofit.
- **인증**: Google 로그인을 통한 Firebase Auth.

### 서버 측 (Node.js - 별도 리포지토리)
- **목적**: Google Places API의 프록시 역할을 하며, 사용자 선호도를 기반으로 필터링 로직을 처리합니다.
- **주요 엔드포인트**:
  - `/api/searchPlace/nearby`: 필터링과 함께 근처 장소를 가져옵니다.
  - `/api/searchPlace/keyword`: 선호도를 포함한 텍스트 기반 검색.
  - `/api/searchPlace/photo-url`: 장소 사진 URL을 검색합니다.
- **필터링 로직**:
  - **가격 필터링**: 선택된 가격 수준(무료, 저가, 보통, 고가)에 따라 장소를 일치시킵니다.
  - **이동 수단 조정**: 검색 반경을 조정합니다(도보: 500m, 자차: 10km).
  - **위치 처리**: 제공된 위경도 또는 서울 기본값과 확장 반경(200km)을 사용합니다.

### 데이터 흐름
1. SettingsScreen에서 사용자 선호도를 설정하고 DataStore에 저장합니다.
2. RecommendationScreen이 ViewModel을 통해 장소를 로드하고, 선호도를 서버로 전송합니다.
3. 서버가 Google Places API를 쿼리하고, 필터를 적용하여 결과를 반환합니다.
4. UI에 결과가 표시되며, 즐겨찾기는 Firestore를 통해 관리됩니다.

## 핵심 로직
### 추천 알고리즘
- **반경 계산**: 빈 시간 기간(예: <30분: 300m, <60분: 600m) 또는 이동 수단 모드를 기반으로 합니다.
- **선호도 필터링**: 서버 측 필터링으로 일치하는 장소만 반환되도록 합니다.
- **위치 폴백**: 위치 비활성화 시 대한민국 전역을 큰 반경(200km)으로 검색합니다.
- **카테고리 매핑**: UI의 칩이 장소 유형에 매핑됩니다(예: "카페" → 카페 관련 유형).

### 설정 로직
- **이동 수단 검증**: 결과가 없어지지 않도록 최소 하나의 이동 수단을 선택하도록 합니다.
- **실시간 업데이트**: StateFlow를 통해 선호도가 UI에 즉시 업데이트됩니다.
- **권한 처리**: 런처를 통한 위치 권한 요청.

### 즐겨찾기 및 AI
- **키워드 생성**: Google Gemini AI를 사용하여 저장된 장소에 대한 검색 가능한 키워드를 생성합니다. 장소 이름, 주소, 평점, 유형을 기반으로 프롬프트를 구성하여 관련 키워드를 추출합니다. 예를 들어, "카페 이름: 스타벅스, 주소: 서울시, 평점: 4.5, 유형: 카페"와 같은 입력으로 "커피, 휴식, 모임" 등의 키워드를 생성합니다.
- **Firestore 통합**: 실시간 리스너가 앱 전체에서 즐겨찾기 상태를 업데이트합니다. 즐겨찾기 추가 시 즉시 키워드 생성을 트리거하며, 실패 시 빈 리스트로 폴백합니다.

### 위치 처리 로직
- **동적 위치 가져오기**: FusedLocationProviderClient를 사용하여 사용자의 마지막 알려진 위치를 가져옵니다. 권한이 부여되고 위치가 활성화된 경우 실제 GPS 좌표를 사용합니다.
- **기본값 폴백**: 위치를 가져올 수 없거나 권한이 거부된 경우 서울 좌표(37.5665, 126.978)를 사용합니다.
- **반경 제한**: Google Places API 제한에 따라 모든 검색 반경을 50,000m로 제한합니다.
- **로깅**: 위치 사용 여부를 로그로 기록하여 디버깅을 지원합니다.

## API 및 통합
### Google Places API
- **역할**: 장소 검색, 세부 정보, 사진 제공. 클라이언트에서 직접 호출하지 않고 Node.js 서버를 통해 프록시하여 API 키 노출을 방지합니다.
- **사용 방식**: 서버에서 Nearby Search, Text Search, Place Photos API를 호출. 요청에 위치, 반경, 키워드, 필터(가격, 이동 수단)를 포함합니다.
- **데이터 제공**: 장소 ID, 이름, 주소, 평점, 가격 수준, 사진 메타데이터, 유형 등을 반환합니다. 필터링 후 최대 20개 장소를 반환합니다.

### Google Gemini AI API
- **역할**: 즐겨찾기 장소에 대한 키워드 생성. AI를 활용하여 장소의 관련성을 높입니다.
- **통합**: BuildConfig에 API 키를 저장하며, Gradle 속성, local.properties, 환경 변수에서 우선순위로 로드합니다. 키가 없으면 키워드 생성을 건너뜁니다.
- **사용 방식**: 즐겨찾기 추가 시 GeminiHelper 클래스를 통해 호출. 장소 데이터를 기반으로 프롬프트를 생성하고, JSON 응답을 파싱하여 키워드 리스트를 추출합니다. 예외 발생 시 빈 리스트 반환.
- **장점**: 수동 태깅 없이 자동으로 검색 가능한 태그를 생성하여 사용자 경험을 향상시킵니다.

### Firebase 연결
- **인증 (Auth)**: Google Sign-In을 통해 사용자 로그인. FirebaseAuth 인스턴스로 현재 사용자 정보를 관리하며, 프로필 사진, 이름, 이메일을 SettingsScreen에 표시합니다.
- **데이터베이스 (Firestore)**: NoSQL 클라우드 데이터베이스로 즐겨찾기 장소를 저장. 문서 구조: placeId, name, address, rating, photoName, imageUrl, types, primaryType, keywords. 실시간 리스너로 즐겨찾기 상태를 동기화합니다.
- **연결 방식**: Firebase SDK를 통해 초기화되며, 사용자별 컬렉션(users/{uid}/wanted)을 사용합니다. 오프라인 지원으로 네트워크 문제 시 로컬 캐시를 활용합니다.

### Google Play Services (위치)
- **FusedLocationProviderClient**: 사용자의 위치를 가져오기 위해 통합. 마지막 알려진 위치를 비동기적으로 검색하며, 권한 확인 후 실행합니다.
- **권한 관리**: ACCESS_FINE_LOCATION 권한을 확인하고, 부여된 경우에만 위치를 요청합니다.
- **에러 처리**: 위치 서비스 실패 시 기본 위치로 폴백하여 앱 안정성을 유지합니다.

- **타사 라이브러리**:
  - Retrofit: HTTP 클라이언트.
  - Coil: 이미지 로딩.
  - DataStore: 선호도 저장소.
  - Compose BOM: UI 컴포넌트.

## 설치 및 설정
### 요구 사항
- Android Studio (최신 버전 권장)
- JDK 17
- Android SDK API 24 이상
- Google Play Services
- Node.js 서버 (별도 설정 필요)

### 앱 빌드
1. 프로젝트를 Android Studio에서 엽니다.
2. `local.properties` 파일에 다음을 추가합니다:
   ```
   GEMINI_API_KEY=your_gemini_api_key_here
   ```
3. Gradle 동기화를 수행합니다.
4. 앱을 빌드하고 실행합니다.

### 서버 설정 (Node.js)
1. 별도 리포지토리에서 서버 코드를 클론합니다.
2. 환경 변수에 Google Places API 키를 설정합니다.
3. 서버를 실행합니다: `npm start`
4. 앱에서 서버 URL을 `http://10.0.2.2:3000` (에뮬레이터) 또는 실제 IP로 설정합니다.

## 사용법
1. 앱을 실행하고 Google 로그인을 수행합니다.
2. 설정에서 카테고리, 이동 수단, 가격을 선택합니다.
3. 위치 권한을 부여하여 개인 맞춤 추천을 받습니다.
4. 추천 화면에서 장소를 검색하고 즐겨찾기에 추가합니다.
